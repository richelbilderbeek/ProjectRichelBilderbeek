(C++) Test/test

 

Testing is the process of validating your code. TDD ('Test Driven Design') is a type of software design techniques [6] that uses testing to incrementally write and change software.

 

There are multiple types of tests, which overlap:

  • regression testing: check that new changes does not break other code.
  • Graphical User Interface testing: check that the GUI behaves as expected

 

Boost.Test is a Boost library for testing.

 

 

 

 

Advice

 

  • Distributing testing throughout the design hierarchy can be much more effective per testing dollar than testing at only the highest-level interface [1]
  • Testing a component in isolation is an effective way to ensure reliability [2]
  • Thorough regression testing is expensive but essential; the appropriate time to create thorough regression tests is tied to the stability of the subsystem to be tested [3]
  • Write the test code first [4,5]
  • Keep the test code with the class to be tested [4]
  • Write the test code to verify that your class design is complete [5]
  • Automate the running of your tests [4]
  • If you can’t write test code, you don’t know what your class looks like [5]
  • The act of writing test code will often show features or constraints you were previously unaware of [5]
