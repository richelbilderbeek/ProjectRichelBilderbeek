(Tool) OpenFOAM

 

OpenFOAM is 'a C++ toolbox for the development of customized numerical solvers, and pre-/post-processing utilities for the solution of continuum mechanics problems, including computational fluid dynamics (CFD)' [1]

 

This page contains my notes about OpenFOAM. OpenFOAM is originally developed under Linux, my need is to get it working under Windows.

I downloaded a Windows installer of OpenFOAM version 1.5 from the Sourceforge project 'openfoam-mswin', which worked conveniently.

OpenFOAM collaborates with other tools:

 

  • OpenFOAM designer (main screen screenshot (png), about screen screenshot (png)) (executable at C:\cfd\OpenFOAM-1.5\bin\ofdesign.exe)
  • ParaView (screenshot (png) (executable at "C:\Program Files\ParaView 4.0.1\bin\paraview.exe")

 

 

 

 

 

Implications of OpenFOAM being developed under Linux for Windows users

 

Linux has case sensitive filenames, where Windows does not. Therefore, OpenFOAM cannot simply be compiled under Windows, as it uses different files that are distinguised by their case, for example 'Vector.H' and 'vector.h'.

 

Luckily, there are some Windows (32 and 64 bit) binaries. The common drawbacks of these appy:

  • DLL Hell
  • Need to set PATH, clashing op versions. This can be easily overcome by writing a shell script that temporarily does so. Some suppliers, like blueCAPE have written such a script for you

  • blockMesh: keyword patches is undefined in dictionary "MyFolder\constant\polyMesh\blockMeshDict"
  • icoFoam: cannot find file "points" in directory "constant\polyMesh"
  • icoFoam: cannot open file 'system\fvSchemes'
  • OpenFOAM designer: not a valid case
  • checkMesh error: the mesh has multiple regions which are not connected by any face (fix)
  • potentialFoam: Unable to set reference cell for field p
  • rhosimplefoam: cannot find equation relaxation factor for 'pEqn' or a suitable default value
  • rhosimplefoam: crash at timestap 15 at example MyFolder\tutorials\compressible\rhoSimplecFoam\squareBend
